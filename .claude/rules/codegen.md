# Code Generation Rules

## HTTP Interface Pipeline

The full pipeline from API definition to Go code is:

```
TypeSpec (.tsp)  →  OpenAPI (openapi.yaml)  →  Go Fiber interface (spec.gen.go)
```

Run everything with one command:
```bash
make gen-all
```

### What gen-all does

1. **TypeSpec compile** — reads `cmd/typespec_openapi_generator/v1/` and writes:
   - `src/interface/fiber_server/route/typespec/openapi.yaml`

2. **Go codegen** — reads `src/interface/fiber_server/route/*/openapi.yaml` via `oapi-codegen` and writes:
   - `src/interface/fiber_server/route/typespec/spec.gen.go`

### TypeSpec source layout

```
cmd/typespec_openapi_generator/v1/
├── main.tsp                        ← service definition + all imports
├── tspconfig.yaml                  ← emit config (output-file: "openapi.yaml")
├── model/
│   ├── api-response.model.tsp      ← ApiResponseModel<T>, ErrorResponse, SuccessResponse
│   ├── order.model.tsp             ← order domain models
│   └── store.model.tsp             ← store domain models
└── route/
    ├── order.route.tsp             ← /order routes
    └── store.route.tsp             ← /company, /store, /store/{id}/order routes
```

**Naming conventions in TypeSpec:**
- Model files: `<domain>.model.tsp`
- Route files: `<domain>.route.tsp`
- Namespace for models: `DemoAPI.Model`
- Namespace for routes: `DemoAPI.Routes`
- Routes use: `@tag("<domain>")`, `@route("/<domain>")`, nested `namespace` for sub-resources

## Files That Must NEVER Be Manually Edited

| File | Generated by |
|------|-------------|
| `src/interface/fiber_server/route/typespec/openapi.yaml` | `make gen-all` (step 1) |
| `src/interface/fiber_server/route/typespec/spec.gen.go` | `make gen-all` (step 2) |
| `src/interface/grpc_server/**/*.pb.go` | `make generate-grpc` |
| `src/interface/grpc_server/**/*_grpc.pb.go` | `make generate-grpc` |

Any manual changes to these files will be overwritten on the next codegen run.

## After Running gen-all

Check `src/interface/fiber_server/route/typespec/route.go` against the updated `ServerInterface` in `spec.gen.go`:
- **New route added** → add stub in `route.go`: `panic("implement me")`
- **Route removed** → delete the corresponding method from `route.go`
- **Model changed** → update conversion functions in `model_<domain>.go`

Then run `make unit-test` to confirm the project compiles.

## gRPC Generation

```bash
make generate-grpc
```

After running: check that service implementations in `*_grpc_server.go` still satisfy the updated generated interface. Add stubs for new RPC methods if needed.
