# Vue setup

## Install

```sh
npm i @sellsuki/sellsuki-ui
```

## Usage

```tsx
// src/main.js
import Vue from "vue";
import App from "./App.vue";

// Import the ui-library package with our ssk-button Web Component
import "sellsuki-components/sellsuki-components.js";
import "sellsuki-components/style.css";

Vue.config.productionTip = false;
new Vue({
  render: (h) => h(App),
}).$mount("#app");
```

```html
<!-- src/App.vue -->
<template>
  <ssk-theme-provider>
    <div id="app">
      <h1>Web Components</h1>
      <!-- Our web component! -->
      <ssk-button>Hello Vue</ssk-button>
    </div>
  </ssk-theme-provider>
</template>

<script>
  export default {
    name: "App",
  };
</script>

<style>
  #app {
    font-family: Avenir, Helvetica, Arial, sans-serif;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
    text-align: center;
    color: #2c3e50;
    margin-top: 60px;
  }
</style>
```

## Known Issues


Warning: Failed to resolve component: ssk-___ 
```
HomeView.vue:36 [Vue warn]: Failed to resolve component: ssk-toggle
If this is a native custom element, make sure to exclude it from component resolution via compilerOptions.isCustomElement. 
  at <HomeView onVnodeUnmounted=fn<onVnodeUnmounted> ref=Ref< Proxy(Object) {__v_skip: true} > > 
  at <RouterView> 
  at <App>
```

This can be fix by adding 'compilerOptions' in 'vite.config.ts'
```javascript
export default defineConfig({
  server: {
    port: 5173,
    host: '0.0.0.0',
  },
  plugins: [
    vue(
      {
        template: {
          compilerOptions: {
            isCustomElement: (tag) => tag.startsWith('ssk-'),
          }
        }
      }
    ),
    vueJsx(),
  ],
})

```
